Sending mails from a WordPress website sometimes receive in the spam folder. To avoid that, you would have to configure SMTP authentication for your website. Assuming you are using WordPress, there are many plugins available to complete this task.

Using the plugin and configuring sometimes won't help you to receive emails and would throw back errors related to authentication or connection timeout. So, I have posted the complete solution below from which you could successfully configure SMTP for your website.

Solution:

Step 1:

Login to your WordPress wp-admin control panel. Click on Plugins and then add-new button as shown in the below figure.

Step 2:

Search for the SMTP keyword. For demo purpose, I searched for wp SMTP and installed a plugin called "Easy WP SMTP" . While installing plugins make sure the plugin is updated regularly by its author and it has good reviews.

Click on the Install button and then Activate button to enable the plugin.

Step 3:

As shown in the figure below, go to plugins and click on the Settings link.

Next, you will see a settings page like below where you have to enter your SMTP details. I'll walk you through each one.

  1. "From Email Address": Enter the email address from which the website send mail.
  2. "From Name": Give a name you wish that should be shown when others receive emails from your website.
  3. "SMTP Host": This is the SMTP server name that you use. For example, if you are using Gmail mail service, then your SMTP host will be "smtp.gmail.com"
  4. "Type of Encryption": Select the TLS radio button.
  5. "SMTP Port": If you select TLS encryption, then the port will be 587 and if you choose SSL then the port will be 465.
  6. "SMTP AUTHENTICATION": Select "yes" radio button,
  7. "SMTP Username": This would be the email id that you use to login to your mail account.
  8. "SMTP Password": Enter your mail account password here.

Now save all the changes by clicking the "Save Changes" button.

It's time to check if your details are correct and you are able to send mail via your SMTP details.

Under the Testing and debugging settings area, enter the details and click on send mail. You would probably encounter with some error like the below

SMTP connect() failed. https://github.com/PHPMailer/PHPMailer/wiki/Troubleshooting or Some Authentication Issue

To avoid that, open your Gmail account and click or copy-paste this link on another tab of your browser.

Select "Turn on" radio button to allow access and again try to send mail via testing and debugging settings. You would be able to send mail.
